PROBLEM #1: TRAINING PROGRAMS PAGE - DATA LEAKAGE
================================================================================

FILE: app/[locale]/(cms)/portal/training/page.tsx
LINES: 34, 50

PROBLEM:
- Two select("*") queries exposed all columns including:
  * created_by (User UUIDs)
  * Internal metadata fields
  * Admin-only status information
- Members could inspect Network tab in DevTools and see creator UUIDs
- Cross-member training registration data exposed

SOLUTION APPLIED:
Line 34 - Training Programs Query:
  BEFORE: .select("*")
  AFTER:  .select("id, title, description, start_date, end_date, location, max_participants, status, visible_tiers, created_at")

Line 50 - Training Registrations Query:
  BEFORE: .select("*")
  AFTER:  .select("id, training_id, status, registration_date")

IMPACT: Prevents UUID leakage, hides admin metadata, restricts to own registrations

PROBLEM #8: ADMIN COLLABORATION PAGE - MISSING AUTHORIZATION + LEAKAGE
================================================================================

FILE: app/[locale]/(cms)/admin/collaboration/page.tsx
LINES: 10-26 (NEW), 29, 35

PROBLEM #8A - Missing Authorization Check:
- Page lacked admin role verification
- Any authenticated user could access collaboration data
- No access control on admin endpoint

Problem #8B - Data Leakage:
- Collaborations query: select("*") exposed unnecessary fields
- Pitches query: select("*, ...") exposed all columns
- Admin overexposure to internal metadata

SOLUTION APPLIED:

Authorization Check Added (NEW - Lines 15-25):
  const { data: adminProfile } = await supabase
    .from("profiles")
    .select("role")
    .eq("id", user.id)
    .single();

  if (adminProfile?.role !== "admin") {
    redirect("/portal/dashboard");
  }

Collaborations Query (Line 29):
  BEFORE: .select("*")
  AFTER:  .select("id, title, description, industry, looking_for, deadline, status, visible_tiers, created_at, created_by")

Pitches Query (Line 35):
  BEFORE: .select("*, profiles(company_name, email)")
  AFTER:  .select("id, collaboration_id, member_id, message, supporting_document_url, status, created_at, profiles(company_name, email)")

IMPACT: Prevents unauthorized access to admin features, reduces field exposure
